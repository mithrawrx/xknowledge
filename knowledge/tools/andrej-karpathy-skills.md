一个100行的 CLAUDE.md 文件，将 Andrej Karpathy 反复吐槽的 LLM 编码坏习惯浓缩成4条铁律，干翻了所有 LLM 编码 prompt。

## 核心功能

- **零依赖零配置**: 没有框架、没有依赖，就一个 CLAUDE.md 文件
- **4条编码铁律**: 浓缩 Karpathy 对 AI 编程代理的核心观察
- **立竿见影**: 扔进项目根目录即可改善 Claude Code 编码质量
- **GitHub Trending 第一**: 一周暴涨 4.4 万星，目前已破 11 万星

## 背景

Andrej Karpathy 曾多次指出 LLM 编程代理的常见坏习惯，包括自信选错路径、过度重构、完成报告但代码无法运行等。这个项目将这些观察提炼成可执行的规则。
